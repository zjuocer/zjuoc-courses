# 海洋学院学习小窝：资料上传与网页更新教程

这份教程写给第一次接触 GitHub、VS Code 和 Markdown 的维护者。目标是让你知道：一个资料文件从电脑本地，怎样放进仓库，怎样写到网页上，怎样检查，最后怎样上传到 GitHub 并等网站自动更新。

## 一、先理解整个网站是怎么工作的

这个网站不是在后台管理系统里点按钮上传的，而是一个 GitHub 仓库生成的静态网站。

你本地电脑上有一个仓库文件夹：

```text
C:\Users\ROG\Desktop\zjuoc-courses
```

这个文件夹里的 `docs` 目录就是网站正文。你改 `docs` 里面的 Markdown 文件，网站文字就会变；你把 PDF、Word、zip 放进 `docs` 下面的 `assets` 文件夹，再在 Markdown 里写链接，网页上就会出现下载入口。

你把本地修改通过 Git 上传到 GitHub 后，GitHub Pages 会自动重新生成网站。过一小会儿，线上地址就会更新：

```text
https://zjuocer.github.io/zjuoc-courses/
```

## 二、每次开始前要做什么

### 1. 打开 VS Code

打开 VS Code 后，选择：

```text
文件 -> 打开文件夹
```

然后选择：

```text
C:\Users\ROG\Desktop\zjuoc-courses
```

不要只打开某一个 Markdown 文件，要打开整个 `zjuoc-courses` 文件夹。这样 VS Code 左侧才会显示完整项目结构，Git 功能也能正常识别。

### 2. 先拉取云端最新内容

在 VS Code 顶部菜单选择：

```text
终端 -> 新建终端
```

终端打开后，先确认当前位置是仓库根目录。如果不是，可以输入：

```powershell
cd C:\Users\ROG\Desktop\zjuoc-courses
```

然后输入：

```powershell
git pull
```

这一步的意思是：先把 GitHub 上别人可能已经更新的内容同步到本地，避免你在旧版本上继续改。

## 三、看懂仓库里的文件夹

仓库里最重要的是这些位置：

```text
zjuoc-courses/
  docs/
    index.md
    1-freshman/
    2-sophomore/
    3-junior/
    4-senior/
    community/
    flying-handbook/
    stylesheets/
  zensical.toml
  README.md
```

各自含义如下：

`docs/index.md` 是网站首页。

`docs/1-freshman/` 是大一课程。

`docs/2-sophomore/` 是大二课程。

`docs/3-junior/` 是大三课程。

`docs/4-senior/` 是大四课程。

`docs/community/` 是社区页面，比如名人堂、个人主页。

`docs/flying-handbook/` 是飞跃手册页面。

`docs/stylesheets/extra.css` 是网站额外样式。一般上传资料不需要改它。

`zensical.toml` 是网站导航配置。新增课程、新增页面时，经常要改这里，否则网页侧边栏可能找不到新页面。

`README.md` 是给仓库维护者看的说明，不是主要网页内容。

## 四、每门课程的标准结构

一门课程通常这样放：

```text
docs/年级目录/课程英文短名/
  index.md
  papers.md
  notes.md
  study-guide.md
  resources.md
  assets/
    papers/
    notes/
    course-materials/
```

例如海洋科学基础：

```text
docs/1-freshman/ocean-science-foundation/
```

例如水下机器人学：

```text
docs/2-sophomore/underwater-robotics/
```

页面文件的含义：

`index.md` 是课程主页，放四个蓝色按钮入口。

`papers.md` 是历年卷 / 回忆卷。

`notes.md` 虽然文件名叫 notes，但网页标题统一叫“课程资源”。这里放笔记、讲义、教材答案、自测题、复习资料等学习资源。

`study-guide.md` 是课程评价。这里放评分细则、学习方法、老师评价、主观体验。

`resources.md` 是课程资料。这里放 PPT、课件、课程大纲、作业集合、课程原始资料等。

资源文件夹的含义：

`assets/papers/` 放历年卷、回忆卷。

`assets/notes/` 放课程资源里的文件，比如笔记、讲义、自测题。

`assets/course-materials/` 放课程资料里的文件，比如 PPT、课件、大纲、作业。

如果有特殊板块，可以新增文件和资源目录。比如水下机器人学有：

```text
midterm.md
assets/midterm/
```

这就是期中汇报板块和对应资料目录。

## 五、上传一个文件的完整流程

假设你要把一个 PDF 上传到海洋科学基础的课程资源板块。

### 1. 判断它属于哪个板块

先判断文件类型和用途：

历年卷、回忆卷放到：

```text
papers.md
assets/papers/
```

笔记、讲义、教材答案、复习笔记、自测题放到：

```text
notes.md
assets/notes/
```

PPT、课件、课程大纲、作业、课程原始资料放到：

```text
resources.md
assets/course-materials/
```

评分细则、学习方法、老师评价放到：

```text
study-guide.md
```

飞跃手册放到：

```text
docs/flying-handbook/index.md
docs/flying-handbook/assets/
```

名人堂文字放到：

```text
docs/community/hall-of-fame.md
```

### 2. 把文件复制到对应 assets 文件夹

比如你要上传：

```text
C:\Users\ROG\Desktop\海洋科学基础个人讲义.pdf
```

放到海洋科学基础课程资源，就复制到：

```text
C:\Users\ROG\Desktop\zjuoc-courses\docs\1-freshman\ocean-science-foundation\assets\notes\
```

复制后路径应该类似：

```text
docs/1-freshman/ocean-science-foundation/assets/notes/海洋科学基础个人讲义.pdf
```

### 3. 打开对应 Markdown 页面

课程资源页面是：

```text
docs/1-freshman/ocean-science-foundation/notes.md
```

在合适位置加入：

```markdown
### 海洋科学基础个人讲义

- [海洋科学基础个人讲义.pdf](<./assets/notes/海洋科学基础个人讲义.pdf>)

贡献者名单：王景麟
```

注意链接写法：

```markdown
[网页上显示的文字](<文件相对路径>)
```

如果文件名里有中文、空格、括号，建议路径外面加尖括号：

```markdown
(<./assets/notes/海洋科学基础个人讲义.pdf>)
```

这样最稳。

## 六、贡献者应该怎么写

本站统一规则是：贡献者写在对应文件或文件组下面，不写在页面最底部的黄色框里。

一个文件一个贡献者时：

```markdown
- [文件A.pdf](<./assets/notes/文件A.pdf>)

贡献者名单：张三
```

连续几个文件都是同一个人贡献时，只需要在最后一个文件下面写一次：

```markdown
- [文件A.pdf](<./assets/notes/文件A.pdf>)
- [文件B.pdf](<./assets/notes/文件B.pdf>)
- [文件C.pdf](<./assets/notes/文件C.pdf>)

贡献者名单：张三
```

如果中间换了贡献者，就要分开写：

```markdown
- [文件A.pdf](<./assets/notes/文件A.pdf>)

贡献者名单：张三

- [文件B.pdf](<./assets/notes/文件B.pdf>)

贡献者名单：李四
```

页面底部黄色框只写更新时间：

```markdown
## 更新记录

<div class="signature-box" markdown>

- 最后更新：2026-07-20 16:50:15 +08:00

</div>
```

## 七、更新时间怎么写

每次改完某个页面，都建议更新这个页面底部的时间。

在 VS Code 终端输入：

```powershell
Get-Date -Format "yyyy-MM-dd HH:mm:ss zzz"
```

会得到类似：

```text
2026-07-20 16:50:15 +08:00
```

把它复制到页面底部黄色框：

```markdown
- 最后更新：2026-07-20 16:50:15 +08:00
```

## 八、新增一门课程怎么做

假设要在大一新增“海洋科学基础”。

### 1. 新建课程文件夹

在：

```text
docs/1-freshman/
```

下面新建：

```text
ocean-science-foundation/
```

课程文件夹名建议用英文小写，用短横线连接，不要用空格。

### 2. 新建页面

在课程文件夹里新建：

```text
index.md
papers.md
notes.md
study-guide.md
resources.md
```

### 3. 新建资源文件夹

在课程文件夹里新建：

```text
assets/
  papers/
  notes/
  course-materials/
```

### 4. 写课程主页

`index.md` 可以按这个模板：

```markdown
# 课程名称

## 资料分支

<div class="branch-grid" markdown>

<div class="branch-card" markdown>

### 历年卷 / 回忆卷

历年回忆卷、题型整理和参考材料。

[进入板块](./papers.md){ .md-button .md-button--primary }

</div>

<div class="branch-card" markdown>

### 课程资源

笔记、讲义、教材答案和复习资料。

[进入板块](./notes.md){ .md-button .md-button--primary }

</div>

<div class="branch-card" markdown>

### 课程评价

评分细则、学习方法、老师评价和复习建议。

[进入板块](./study-guide.md){ .md-button .md-button--primary }

</div>

<div class="branch-card" markdown>

### 课程资料

PPT、课件、课程大纲和后续补充资料。

[进入板块](./resources.md){ .md-button .md-button--primary }

</div>

</div>
```

### 5. 修改年级总览页

如果课程在大一，就改：

```text
docs/1-freshman/index.md
```

在课程列表里加卡片：

```markdown
<div class="course-card" markdown>

### 海洋科学基础

已创建课程主页、历年卷、课程资源、课程评价、课程资料等板块。

[进入课程](./ocean-science-foundation/){ .md-button .md-button--primary }

</div>
```

### 6. 修改导航

打开：

```text
zensical.toml
```

找到对应年级，在导航里加课程：

```toml
{ "海洋科学基础" = [
  { "课程主页" = "1-freshman/ocean-science-foundation/index.md" },
  { "历年卷/回忆卷" = "1-freshman/ocean-science-foundation/papers.md" },
  { "课程资源" = "1-freshman/ocean-science-foundation/notes.md" },
  { "课程评价" = "1-freshman/ocean-science-foundation/study-guide.md" },
  { "课程资料" = "1-freshman/ocean-science-foundation/resources.md" },
] },
```

如果不改 `zensical.toml`，网页可能能直接访问新页面，但侧边栏不会出现入口。

## 九、zip 大文件怎么处理

GitHub 普通仓库有一个重要限制：单个文件不能超过 100MB。

如果 zip 小于 100MB，可以直接上传。

如果 zip 超过 100MB，不要硬推。常见做法有两种：

第一种：拆成分卷，每个分卷小于 100MB，然后在网页上列出所有分卷。

第二种：把大文件放到网盘或 GitHub Release，再在网页上放外部链接。

本仓库这次使用的是第一种。网页写法类似：

```markdown
- [期中.zip 分卷 001](<./assets/midterm/midterm-2025-2026-wjl.zip.001>)
- [期中.zip 分卷 002](<./assets/midterm/midterm-2025-2026-wjl.zip.002>)
- [期中.zip 分卷 003](<./assets/midterm/midterm-2025-2026-wjl.zip.003>)

贡献者名单：王景麟
```

同时要告诉下载者怎么合并：

```powershell
cmd /c copy /b midterm-2025-2026-wjl.zip.001+midterm-2025-2026-wjl.zip.002+midterm-2025-2026-wjl.zip.003 期中.zip
```

合并后得到的 `期中.zip` 才是原始压缩包。


```text
docs/flying-handbook/assets/
```

## 十二、本地检查

改完后，先在终端运行：

```powershell
.\.venv\Scripts\zensical.exe build --clean
```

如果看到类似：

```text
Build finished
No issues found
```

说明网站可以正常生成。

如果报错，通常是这些原因：

文件链接写错了。

文件实际没有复制到对应位置。

Markdown 里的括号、尖括号少了一个。

`zensical.toml` 导航格式写错了，比如少了逗号或括号。

## 十三、查看修改了哪些文件

输入：

```powershell
git status --short
```

常见符号含义：

`M` 表示修改过的文件。

`A` 表示新添加的文件。

`D` 表示删除的文件。

`??` 表示 Git 还没有跟踪的新文件。

如果看到不认识的文件，先不要提交，确认它是不是这次要改的内容。

比如 `tmp/`、`output/` 这种临时目录，一般不要提交。

## 十四、暂存、提交、推送

### 1. 暂存本次要上传的文件

如果你确认所有修改都是这次要上传的，可以用：

```powershell
git add docs zensical.toml
```

如果仓库里有其他人的临时改动，最好精确添加本次文件。例如：

```powershell
git add docs\2-sophomore\underwater-robotics\midterm.md
git add docs\2-sophomore\underwater-robotics\assets\midterm
git add docs\flying-handbook\index.md
git add docs\flying-handbook\assets\飞跃手册2024海院.docx
git add docs\community\hall-of-fame.md
git add docs\1-freshman\ocean-science-foundation\notes.md
git add docs\1-freshman\ocean-science-foundation\assets\notes\海洋科学基础个人讲义.pdf
```

### 2. 提交

提交就是给这次改动拍一个快照，并写一句说明：

```powershell
git commit -m "更新期中资料和飞跃手册"
```

说明不用很长，但要能看懂这次改了什么。

### 3. 推送

推送就是把本地提交上传到 GitHub：

```powershell
git push origin main
```

推送成功后，GitHub Pages 会自动重新构建网站。

## 十五、线上检查

推送后不要马上以为网站已经更新。GitHub Pages 通常需要几十秒到几分钟。

你可以打开这些地址检查：

```text
https://zjuocer.github.io/zjuoc-courses/
```

如果是课程页，就打开对应页面。例如：

```text
https://zjuocer.github.io/zjuoc-courses/2-sophomore/underwater-robotics/midterm/
```

如果页面还没变，先等一分钟刷新。不要反复乱改。

## 十六、最容易犯的错误

第一，文件复制了，但 Markdown 没写链接。这样网页上看不到。

第二，Markdown 写了链接，但文件没复制。这样网页会出现打不开的链接。

第三，文件名和链接不完全一致。中文、空格、括号都必须一样。

第四，新增课程后忘记改 `zensical.toml`。这样侧边栏没有入口。

第五，把贡献者写到页面最底部黄色框里。本站规则是贡献者跟着文件走，黄色框只写更新时间。

第六，把临时目录也提交了。提交前一定看 `git status --short`。

第七，大文件超过 100MB 还直接提交。超过 100MB 的文件要拆分、压缩或改用外部链接。

## 十七、推荐的每次更新顺序

每次更新可以按这个顺序来：

1. 打开 VS Code 整个仓库文件夹。
2. 在终端运行 `git pull`。
3. 判断资料应该放在哪个板块。
4. 把文件复制到对应 `assets` 文件夹。
5. 修改对应 Markdown 页面，添加下载链接、学年、贡献者。
6. 更新页面底部黄色框时间。
7. 如果新增课程或新增页面，修改 `zensical.toml`。
8. 运行 `.\.venv\Scripts\zensical.exe build --clean`。
9. 运行 `git status --short` 检查文件。
10. 用 `git add` 暂存本次文件。
11. 用 `git commit -m "说明"` 提交。
12. 用 `git push origin main` 推送。
13. 等 GitHub Pages 更新后，打开线上页面检查。

按照这个流程走，基本就能稳定完成“本地资料上传到云端网站”的全流程。
